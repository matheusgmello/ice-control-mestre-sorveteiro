  estavam dessincronizadas com os dados reais. Isso é comum quando dados são inseridos manualmente ou por scripts que
  não atualizam o contador automático do PostgreSQL.

      (vendas, pagamentos, itens, produtos, etc.) para o valor máximo atual + 1.
   2. Prevenção: Adicionei um endpoint oculto na API (/api/fix-sequences) que pode ser usado no futuro caso esse
      problema ocorra novamente após uma migração de dados.
